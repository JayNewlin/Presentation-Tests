TOUCH EVENT SUPPORT

Faster Click Events

Standard "onclick" JavaScript events can be used on touch based iOS devices to respond to finger taps. Unfortunately, since iOS devices respond to many other types of touch events (double taps, swipes, pinch zooms, etc.), there is about a 300ms delay in the execution of "onclick" events while the system verifies that the original tap wasn't intended to begin some more complicated gesture.

Mobile developer Matteo Spinelli solved the click delay problem in a JavaScript helper library, NoClickDelay, that more effectively processes the available touch events and executes the "onclick" event in response to a more appropriate sequence of touch events. It is strongly encouraged that all Harmony Viewer content use NoClickDelay or some similar script to expedite "onclick" execution.

Scrollable Content Areas

It's often beneficial to define one or more areas of the slide with self-contained scrollable content blocks. Mobile developer Matteo Spinelli solved this content scrolling problem in a third party JavaScript helper library, iScroll, which enables singer finger scrolling of an HTML div element of fixed dimensions. It is strongly encouraged that all Harmony Viewer content use iScroll or some similar script to implement scrollable content areas within slide content.
